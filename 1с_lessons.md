## 1С записки
Как раньше хранились константы? В виде таблицы (условно)  Таблица: Константы; Поля: НаименованиеПоля, Значение
Как сейчас хранится? Таблица:НаименованиеКонстанты; Поля: Значение
те для каждой констаныт своя таблица. Это сделано во для решения проблемы блокировок

В константах можно хранить настройки(логины для внешних систем)

Общая форма констант, хранится в дереве, в Общие Формы.
Для Установки Своей формы нужно открыть свойство Конфигурации --> основнаяФормаКонстант

Свойство Конфигурации:
Основной режим запуска: 
  + Обычный 
  + управляемый

Функции\

Структуры данных:
1. Массив (редко используется)
2. Список значений (Состоит из 4х неизменяемых стбцов - значение, представление, булево, картинка)
3. Таблица значений (можно навесить индексы бд для ускорения поиска)
4. Дерево значений (очень редко используется)
5. Структура (Ключ: Значение)

Создание резервной копии: Администрирование --> Загрузить/Выгрузить информационную базу. есть проблемы с резервным ручным копированием в процессе работы с веб интерфейсом. Правильным вариантом будет настроить ежедневное копирование на уровне базы данных через sql бд админов

Если не получается войти в базу 1С, надо сначала попробовать сбросить локальный кэш, те удалить и заново добавить базу. Еще вариант через Администрирование --> Тестирование и исправление
Или найти в папке 1с chdbfl.exe и добавить базу (файд с расширением 1CD)

Справочники
Основные атрибуты:
- Ссылка
- Код
- Наименование (представление)
- ПометкаУдаления (булево значение)
- Родитель (при иерархии) - ссылка на родительский элемент
- ЭтоГруппа (булево значение) - указывает значение является элементом или группой
- Владелец

Табличные части используются редко, либо регистры сведений, либо подчиненные справочники 

Вкладка Реквизиты - это источники данных для формы
